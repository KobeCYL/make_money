# 技术规范和开发指南提示词模板

## 1. 代码规范提示词

```
请为[项目名称]创建一个代码规范指南，包含以下部分：

1. 通用规范
   - 命名规范
     - 文件命名
     - 变量命名
     - 函数命名
     - 类命名
   - 注释规范
     - 文件注释
     - 函数注释
     - 代码注释
   - 格式规范
     - 缩进
     - 空行
     - 换行
     - 括号

2. 前端代码规范
   - HTML规范
     - 标签使用
     - 属性顺序
     - 语义化
   - CSS规范
     - 选择器
     - 属性顺序
     - 媒体查询
   - JavaScript规范
     - 变量声明
     - 函数定义
     - 模块组织
   - 组件规范
     - 组件结构
     - 属性定义
     - 生命周期

3. 后端代码规范
   - 接口规范
     - 命名规则
     - 参数规范
     - 响应格式
   - 数据库规范
     - 表命名
     - 字段命名
     - SQL规范
   - 业务逻辑规范
     - 异常处理
     - 日志记录
     - 事务处理

4. 版本控制规范
   - 分支管理
   - 提交信息
   - 合并策略
   - 版本号规则
```

## 使用说明

1. 根据项目需求，选择相应的规范提示词
2. 将[项目名称]替换为实际项目名称
3. 根据项目特点，调整或补充模板内容
4. 确保规范内容清晰、可执行、易于遵守
5. 建议结合实际案例，提供具体的示例说明

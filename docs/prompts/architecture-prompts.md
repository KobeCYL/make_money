# 项目架构设计提示词模板

## 1. 系统架构设计提示词

```
请为[项目名称]创建一个系统架构设计方案，包含以下部分：

1. 架构概述
   - 系统目标
   - 架构原则
   - 技术选型依据
   - 系统边界

2. 系统分层
   - 展示层
     - 用户界面
     - 前端框架
     - 状态管理
   - 应用层
     - 业务逻辑
     - 服务编排
     - 数据处理
   - 数据层
     - 数据模型
     - 存储方案
     - 缓存策略

3. 核心组件
   - 组件职责
   - 组件关系
   - 接口定义
   - 数据流转

4. 技术架构
   - 开发框架
   - 中间件
   - 数据库
   - 缓存系统
   - 消息队列

5. 部署架构
   - 部署模型
   - 服务器配置
   - 网络拓扑
   - 负载均衡

6. 安全架构
   - 认证授权
   - 数据加密
   - 访问控制
   - 安全审计

7. 性能设计
   - 性能目标
   - 扩展策略
   - 优化方案
   - 监控方案
```

## 使用说明

1. 根据项目规模和需求，选择合适的架构设计提示词
2. 将[项目名称]替换为实际项目名称
3. 根据项目特点，调整或补充模板内容
4. 确保设计方案的可行性和可扩展性
5. 建议结合具体业务场景，补充详细的设计说明和示例
